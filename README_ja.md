## 概要

光学ビードコンピューティング（OBQC）は、日本のそろばんの構造と動作ロジックから着想を得た、多値フォトニック情報処理の提案フレームワークです。

そろばんでは、数字は単なる二値列ではなく、珠の配置として表現されます。熟練者は、特にフラッシュ暗算において、数字を逐次的に読むのではなく、配置全体をパターンとして認識・操作します。

OBQCは、この原理を光に転用します。情報を、波長、偏光、位相、時間ビン、パルス幅、空間モード、強度、RGBW/SCDシンボルパターンなど、光の識別可能な複数の自由度にまたがる光学ビードパターンとして表現する構想です。

目的は、光計算が常に優れていると主張することではありません。目的は、パターンベースの光符号化が、光パターン認識、コンパクトな状態表現、多値シンボル復号、高次元光分類などの特定タスクに有効かどうかを、明確・再現可能・反証可能な形で検証することです。

## 概念的起源：そろばんから光学ビードへ

そろばんは、棒上の珠配置として数値を表現します。各棒は、構造化された空間パターンによって10進数字を符号化します。

OBQCは、この考え方を光情報処理へ拡張します。

| そろばん | 光学ビードコンピューティング |
|---|---|
| 棒上の物理的な珠 | 光の自由度 |
| 珠の位置 | その自由度内の状態値 |
| 全体の珠配置 | 光学ビード状態ベクトル |
| フラッシュ暗算 | 光パターン認識 |
| そろばん計算 | 光状態変換 |

重要なのは、完全なパターンそのものが認識単位になるという点です。

## 光学ビード状態モデル

光学ビード状態は、次のように表現できます。

```text
B = (lambda, P, phi, tau, w, s, l, A)
```

各要素は次を意味します。

- `lambda` = 波長 / 色チャンネル
- `P` = 偏光
- `phi` = 位相
- `tau` = 時間ビン
- `w` = パルス幅
- `s` = 空間モードまたは位置
- `l` = 軌道角運動量
- `A` = 振幅または強度

理論上の状態数は、各自由度のレベル数の積です。

```text
N_theoretical = n_lambda × n_P × n_phi × n_tau × n_w × n_s × n_l × n_A
```

ただし、実用的に使える状態数は、常に理論上限より小さくなります。

実用的な状態分離性は、次に依存します。

- 検出器のスペクトル分解能
- 信号対雑音比
- 光学クロストーク
- 熱ドリフト
- 校正安定性
- 背景光
- 検出器飽和
- 位相と偏光の不安定性
- 物理的アライメント

## SCD / RGBW / CMOS 光シンボル試作経路

実用的な近期経路として、次の組み合わせがあります。

```text
Soroban-Coded Decimal（SCD）
+ RGBW光チャンネル
+ LEDまたは表示出力
+ CMOS / カラーセンサー読み出し
+ 最近傍復号
```

SCDは、そろばんの珠構造に着想を得た制約付き5ビットパターンで10進数字を表現します。

```text
D = (H, L4, L3, L2, L1)
```

各要素は次を意味します。

- `H` = 五の珠相当
- `L4, L3, L2, L1` = 一の珠相当

RGBW × SCD符号化では、ゼロ状態の衝突を避けるために、マーカー bit が必要です。

```text
[M, H, L4, L3, L2, L1]
```

マーカーなしでは、R0、G0、B0、W0がすべて同じゼロベクトルに崩壊します。

## エラーモデルと状態分離性

現実の光学システムは、多くの誤差要因の影響を受けます。

- ガウス型センサーノイズ
- ショットノイズ
- 読み出しノイズ
- 暗電流
- 背景光
- チャンネルクロストーク
- 検出器量子化
- 光学アライメント誤差
- 温度ドリフト
- 飽和
- 経年劣化と校正ドリフト

シミュレーターは、これらの効果を近似し、復号精度がどう変化するかを確認するためのものです。

主要な評価指標：

- シンボルエラー率
- 混同行列
- 最近傍マージン
- 棄却率
- 信号対雑音比への感度
- 状態分離しきい値

## 最小プロトタイプロードマップ

### Phase 0：ソフトウェアシミュレーション

- 光学ビードアルファベットを定義する
- 設定可能なノイズを注入する
- 最近傍分類で復号する
- エラー率とマージンを測定する
- バイナリ方式と比較する

### Phase 1：RGBW / CMOSプロトタイプ

- RGBまたはRGBW LEDを使う
- CMOSまたはカラーセンサーで読み出す
- 小規模な状態アルファベットをテストする
- 校正安定性を確認する
- 制御照明下でシンボルエラー率を測定する

### Phase 2：多自由度光プロトタイプ

- 波長、偏光、時間、空間状態符号化を追加する
- 複数自由度を組み合わせてテストする
- 冗長性と棄却しきい値を追加する
- 通常のバイナリ光符号化と比較する

### Phase 3：長期的な量子互換研究

## 制約と限界

重要な制約は次の通りです。

- 実証済み物理プロトタイプはまだ存在しない
- ソフトウェアノイズモデルは単純化されている
- ハードウェアのエネルギーコストは未確定
- 検出器と校正には限界がある
- 光損失がある
- 状態クロストークがある
- 位相と偏光は不安定化しやすい
- 理論上の状態数は、実用状態数を上回る
- ハードウェアベンチマークなしに過剰主張するリスクがある
