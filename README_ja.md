## 概要

光学ビードコンピューティング（OBC）は、日本のそろばんの構造と動作ロジックから着想を得た、多値フォトニック情報処理のための提案フレームワークです。そろばんでは、数値は個々の桁としてではなく、空間的な珠の配置として表現されます。意味は線形なシンボル列ではなく、配置そのものによって運ばれます。

本フレームワークは、この原理を光に応用することを提案します。情報は複数の同時自由度（波長・偏光・位相・時間ビン・パルス幅・空間モード・軌道角運動量）にまたがる*光学ビードパターン*として符号化でき、計算や認識タスクはこれらの多次元光学状態を変換・復号することで実行できます。

フレームワークは3層を区別します。近期の古典多値光学符号化を用いる*決定論的*層、量子ビット研究から高次元状態の概念を借用した*量子着想*層、そして単一光子源と量子ゲートを用いる長期的な*量子光学*拡張です。近期の決定論的層が本リポジトリの主要な焦点です。

本文書は、概念モデル、状態形式論、エラーの考慮事項、最小シミュレーションフレームワーク、およびプロトタイプロードマップを提示します。主張は技術的に根拠のある範囲に限定されています。目標は、そろばん着想フォトニックコンピューティングのための明確で再現可能かつ反証可能な研究フレームワークを提供することです。

## 概念的起源：そろばんから光学ビードへ

参照: [docs/soroban-origin.md](docs/soroban-origin.md)

### そろばん

そろばん（日本の計算器）は、棒に沿った珠の配置として数値を表現します。各棒は位取りの列に対応し、棒内の各珠配置は0から9の数字を符号化します。完全なそろばんの値は、独立に復号されるシンボルの文字列ではなく、すべての棒にわたる珠の配置である*空間パターン*です。

熟練したそろばんの使い手は、数字を逐次的に読みません。全パターンを一度に知覚し操作します。

### フラッシュ暗算

*フラッシュ暗算*では、実践者は画面に数分の1秒間表示される数列を見て、高い精度でその合計を計算します。研究によれば、上級者は仮想のそろばんを頭の中でシミュレートしており、シンボルの桁表現ではなく空間パターンイメージを操作しています。

これは、数値演算がパターンそのものが計算単位となる、急速な*視覚パターン認識と変換*として実行できることを示しています。

### そろばんから光学ビードへ

光学ビードコンピューティングは、この原理をフォトニック情報処理に転用します：

- そろばんが物理的な棒にわたる珠の配置として数値を符号化するのに対し、OBQCは光の自由度にわたる多次元光学配置として状態を符号化します。
- そろばんの使い手が珠パターンを認識・変換するのに対し、OBQCのデコーダーが光学パターンを認識し出力値にマッピングします。
- フラッシュ暗算が人間の認知においてパターン速度の優位性を示すのに対し、OBQCはパターンベースの光学符号化がフォトニックシステムにおいて構造的優位性を提供するかどうかを探究します。

中心的なアナロジー：

| そろばん | 光学ビードコンピューティング |
|---|---|
| 棒上の物理的な珠 | 光の自由度（λ, P, φ, …） |
| 珠の位置（上/下） | その自由度内の状態値 |
| 完全な珠の配置 | 光学ビード状態ベクトル B |
| そろばんの計算 | 光学状態変換 |
| フラッシュパターン認識 | 光学パターン復号 |

## なぜ単なるバイナリフォトニクスではないのか

従来のデジタルエレクトロニクスと標準的な光通信システムは、情報層では根本的にバイナリです。各チャンネルは0または1を運びます。高スループットの光ファイバーシステムでさえ、通常は単一の光学特性を切り替えることで情報を符号化します（パルスの有無、または2状態間の位相シフト）。

OBQCは異なる構造を提案します：

| システム | 情報層 | シンボルあたりの状態数 |
|---|---|---|
| バイナリエレクトロニクス | 配線1本あたり1ビット | 2 |
| 従来の光通信 | オン/オフキーイングまたはBPSK | 2 |
| 高度な光変調（QAM、OFDM） | シンボルあたり複数ビット | 4〜1024以上 |
| **光学ビードコンピューティング** | **多次元珠状態ベクトル** | **設定可能；使用可能な自由度に依存** |

この区別は、生の状態数の問題が主なのではありません。構造的な問題です。OBQCは、完全な多次元光学配置を、独立に多重化されたバイナリチャンネルのセットとしてではなく、そろばんの珠配置に類似した単一の*パターン単位*として扱うことを提案します。

この構造的な違いは以下の点で優位性をもたらす可能性があります：
- 完全な状態ベクトルが意味のある単位となるパターン指向の認識タスク
- 高次元のクラスやテンプレートのコンパクトな符号化
- 複数の自由度にわたる同時動作がチャンネルごとの帯域要件を削減する光学システム

これらの構造的優位性が実用的な計算上の利点に転換されるかどうかは、物理的な実現可能性に依存する未解決の研究課題です。

## 中心仮説

> 光の複数の自由度にわたって識別可能な光学状態を確実に符号化・変換・復号できるならば、多値光学ビードコンピューティングは、バイナリ光学システムとは構造的に異なる、コンパクトで並列的かつパターン指向の情報処理をサポートできる可能性がある。

この仮説は反証可能かつテスト可能です。主要な変数は：

- 現実的なノイズ条件下で、各自由度あたり確実に識別可能な状態の数
- 複数の自由度にわたる共同符号化の関数として達成可能な総アルファベットサイズ
- 状態数・ノイズレベル・冗長符号化の関数としての復号精度
- パターンベースの符号化がバイナリ符号化に対して構造的利点をもたらす計算タスク

## 光学ビード状態モデル

参照: [docs/optical-bead-state-model.md](docs/optical-bead-state-model.md)

### 状態ベクトル

光学ビード状態はタプルとして定義されます：

```
B = (λ, P, φ, τ, w, s, ℓ)
```

各成分は光の1つの自由度を表します：

| シンボル | 自由度 | 状態の例 |
|---|---|---|
| λ | 波長 / カラーチャンネル | 450 nm, 532 nm, 633 nm, 780 nm, … |
| P | 偏光状態 | H, V, D, A, R, L |
| φ | 位相状態 | 0, π/4, π/2, 3π/4, π, … |
| τ | 時間ビン / 時間的位置 | ビン0, ビン1, ビン2, … |
| w | パルス幅 | 狭い, 中間, 広い |
| s | 空間モード / 位置 | モード0, モード1, モード2, … |
| ℓ | 軌道角運動量（OAM） | ℓ = 0, ±1, ±2, … |

### 理論的vs実用的状態数

理論的な状態数は、各自由度のレベル数の積です：

```
N_theoretical = n_λ × n_P × n_φ × n_τ × n_w × n_s × n_ℓ
```

例えば、4波長 × 4偏光 × 4位相 × 4時間ビン = 256理論的状態。

**ただし、実用的に識別可能な状態数は常に理論的最大値より少なくなります。**

実用的な状態分離性は以下に依存します：
- 検出器のスペクトル分解能と感度
- 信号対雑音比
- 各自由度における隣接状態間のクロストーク
- チャンネルドリフトと環境不安定性
- キャリブレーション精度と安定性
- 自由度間の相互作用効果（例：偏光依存位相シフト）

実用上使えるアルファベットサイズは、各物理実装に対して経験的に決定する必要があります。本リポジトリのシミュレータは、ガウスノイズモデル下での分離性推定ツールを提供します。

## 決定論的OBQCと量子OBQC

参照: [docs/deterministic-vs-quantum.md](docs/deterministic-vs-quantum.md)

| 層 | 説明 | ステータス |
|---|---|---|
| **決定論的光学ビードコンピューティング** | 巨視的な光源と検出器を使用した古典多値光学状態符号化・復号 | 近期シミュレーション / プロトタイプ |
| **量子着想OBQC** | 完全な量子コヒーレンスやエンタングルメントを必要とせずに、量子ビット研究の高次元符号化概念を使用 | 概念的 / 研究フレームワーク |
| **量子光学ビードコンピューティング** | 単一光子・量子ビット・エンタングルメント・量子ゲートを使用する将来の拡張 | 長期研究方向 |

決定論的層が主要な近期の焦点です。量子コヒーレンスを必要とせず、以下を使用してプロトタイプ化できます：
- ダイオードレーザーまたはLED
- 標準的な偏光光学素子
- 光学変調器
- カラーセンサーまたは分光器
- 標準Pythonベースの復号ソフトウェア

量子拡張は長期的な研究方向であり、決定論的プロトタイプの前提条件ではありません。

## エラーモデルと状態分離性

参照: [docs/limitations.md](docs/limitations.md)

現実的な光学システムは複数のノイズ源とエラー源に晒されます。シミュレータは以下をモデル化します：

| エラー源 | 説明 | モデル化方法 |
|---|---|---|
| **ガウスノイズ** | 測定された光学パラメータのランダムな変動 | 各次元への加法的ガウスノイズ |
| **検出器量子化** | 有限の検出器分解能が識別可能なレベルを制限 | 床/天井量子化 |
| **チャンネルドリフト** | 時間的な光学特性の緩やかな系統的ドリフト | 線形ドリフト項 |
| **スペクトルオーバーラップ** | 隣接波長チャンネルが相互に漏れ込む | ガウスのスペクトル相互結合 |
| **偏光エラー** | 偏光状態における回転または楕円率誤差 | 角度回転ノイズ |
| **位相不安定性** | 熱的または機械的擾乱による位相変動 | 位相ノイズ項 |
| **時間ジッター** | パルス到着時刻または時間ビン割り当ての不確実性 | ジッター分布 |

**復号戦略：**

デフォルトのデコーダーは正規化された状態空間における最近傍距離を使用します。デコーダーは各受信状態を定義されたアルファベット内の最も近い状態に割り当てます。

**評価指標：**

- アルファベットサイズとノイズレベルの関数としてのシンボルエラーレート（SER）
- 混同行列：どの状態がどの状態と最も混同されやすいか
- 分離性マージン：隣接する状態間の最小距離

- 前向きエラー訂正の統合
- 再現可能なキャリブレーションと測定文書化

### フェーズ3：量子ビット互換研究拡張（長期）

- 単一光子源（例：SPDCまたは量子ドット）
- 時間ビンまたは周波数ビン量子ビット符号化
- 経路符号化と干渉計ゲート
- 量子ゲート操作との互換性評価
- **注記：** このフェーズは決定論的プロトタイプには必要ではなく、独立した長期研究方向を表します。

## 光学媒体の安定化

参照: [docs/optical-medium-stabilization.md](docs/optical-medium-stabilization.md) | [diagrams/optical-medium-options.md](diagrams/optical-medium-options.md) | [simulator/optical_medium_comparison.py](simulator/optical_medium_comparison.py)

OBQCは開放空気光路を必要としません。近期のプロトタイプでは、光学ビードチャンネルをより制御された媒体で封止または置換することで安定化できます。

| 媒体 | 近期の実現可能性 | 主な利点 | 主なリスク |
|---|---|---|---|
| 開放空気 | 非常に容易 | 最も単純な最初のテスト | 粉塵・湿度・乱流 |
| 密封空気 / 窒素 | 容易 | 粉塵と湿度の排除 | 温度と圧力のドリフト |
| 密封液体（水・シリコンオイル） | 中程度 | 封止された屈折率安定化された光路 | 気泡・熱的屈折率ドリフト・位相不安定性 |
| アクリル / PMMAブロック | 容易〜中程度 | 固体・コンパクト・対流なし | 応力複屈折（偏光DOFには不向き） |
| 光学ガラス / 石英 | 中程度 | 低応力・最良の光学品質 | 機械加工コスト |
| ファイバー / 導波路 | 高度 | 誘導型・安定・スケーラブル | 結合の複雑性 |

主要な考慮事項：
- **超純水**は脱気・密封・温度制御が必要な密封液体媒体です。固体材料ではありません。
- **アクリル / PMMA**は波長・強度・空間符号化に適していますが、偏光DOFを劣化させる著しい応力複屈折があります。
- **ガラスと石英**は偏光や位相符号化が必要な場合に推奨されます。
- **ファイバーと導波路**はスケーラブルな実装への長期的な経路です。
- **量子光学拡張**は古典的OBQCプロトタイプより厳格な損失・散乱・位相コヒーレンス制御を必要とします。液体とポリマー媒体は自動的に適切ではありません。

## ハイブリッド量子サポートレイヤー

参照: [docs/hybrid-quantum-support-layer.md](docs/hybrid-quantum-support-layer.md) | [simulator/hybrid_quantum_support_energy.py](simulator/hybrid_quantum_support_energy.py) | [simulator/pattern_vs_binary_operation_cost.py](simulator/pattern_vs_binary_operation_cost.py)

OBQCは超伝導量子プロセッサを置き換えたり、極低温冷却要件を排除したりすることを主張しません。超伝導量子ビットは動作するためにミリケルビン温度を必要とし、いかなる光学補助レイヤーもこれを変えません。

しかし、OBQCはハイブリッド量子システムの周囲インフラのための潜在的な低エネルギーパターン認識サポートレイヤーとして探究できます。多くの近期の量子コンピュータは、量子プロセッサと、制御エレクトロニクス・読み出し識別・シンドローム復号・後処理を含む相当な室温古典インフラを組み合わせたハイブリッドシステムです。

OBQC的な光学パターン認識レイヤーは以下を潜在的に削減できます：
- 読み出し信号分類のオーバーヘッド（読み出し信号をノイズの多い多次元パターンとして扱う）
- 読み出しエレクトロニクスと古典デコーダー間のデータ移動
- 冷凍機へのI/Oチャンネルの高次元多重化
- エラー候補検出における繰り返しパターンマッチングタスクのための繰り返しバイナリ操作
- 選択された古典後処理ワークロードのエネルギーコスト

**このようなレイヤーの正味の利点は条件付きです。** 光学レイヤーが消費するエネルギーが補助処理で節約されるエネルギーより少ない場合にのみ有用です。

| ターゲット | OBQCの役割 | 制限 |
|---|---|---|
| 超伝導量子システム | 補助読み出し / 前処理レイヤー | 極低温量子ビット冷却を除去しない |
| フォトニック量子システム | より自然な光学状態サポートレイヤー | 検出器と損失の制約が残る |
| 古典後処理 | 繰り返しワークロードのパターンベース削減 | オーバーヘッドが節約された計算より低い場合のみ有用 |
| エラー候補検出 | 繰り返しエラーパターンの高速認識 | デジタルデコーダーに対するベンチマーク検証が必要 |
| I/O多重化 | チャンネルあたりの高次元光学符号化 | 統合と変換のオーバーヘッドを特性評価する必要あり |

トイエネルギーモデル（`simulator/hybrid_quantum_support_energy.py`）は、5つのシナリオ（保守的・中程度・楽観的・高OBQCオーバーヘッド・利益なし）にわたって正味利益条件を評価します。

## 密封液体光学ビード媒体

参照: [docs/sealed-liquid-optical-bead-medium.md](docs/sealed-liquid-optical-bead-medium.md) | シミュレータ: [simulator/liquid_medium_noise.py](simulator/liquid_medium_noise.py)

近期の古典OBQCプロトタイプでは、光チャンネルを開放空気にする必要はありません。**密封透明液体媒体**（液浸安定化光学セル）は、粉塵汚染・空気乱流・湿度変動を排除することで環境安定性を改善できます。

このモデルでは、光学ビード状態は閉じた透明液体セル（例：蒸留水・グリセロール溶液・光学グレードオイルで満たした分光キュベット）を通じて伝送されます。液体は機械的に安定した密封された伝播媒体を提供します。

### 潜在的な利点

- 粉塵と空気中の微粒子汚染：密封により排除
- 空気乱流（熱陽炎）：排除；ゆっくりした液体対流に置換
- 湿度変動：密封セル内で排除
- 窓ガラスへの屈折率マッチング：セル境界での反射損失を低減

### 既知の制限

| 制限 | 物理的メカニズム | 深刻度 |
|---|---|---|
| 吸収損失 | ビール-ランベルト減衰：I = I₀ exp(−αL) | 低い（可視波長で5cm水の場合 < 1 dB） |
| 気泡 / 不純物からの散乱 | ランダムな振幅降下 | 低〜中程度（脱気と濾過で排除） |
| 温度依存屈折率ドリフト | dn/dT ≈ −1×10⁻⁴ K⁻¹（水） | **位相符号化にとって重大** |
| 波長依存透過率 | 波長チャンネルごとに異なるα | 中程度；チャンネルごとのキャリブレーションが必要 |
| 長期材料劣化 | 酸化・微生物増殖・ガス放出 | 低い（化学的に安定な媒体と密封容器を使用） |
| 温度による位相不安定性 | 5cm光路で0.1K当たり約5 radの位相シフト | **深刻；以下参照** |

### 重要な発見：センチメートルスケールの光路長では位相符号化は非実用的

シミュレーション（`simulator/liquid_medium_noise.py`）は、5cm光路長の水において、わずか0.1Kの温度偏差が約**5 rad**の位相ドリフトを引き起こすことを示しています。これは位相符号化された光学ビード状態を完全にスクランブルするのに十分です。5cmで < 0.1 radの位相安定性を達成するには、2 mK未満の温度制御が必要であり、近期プロトタイプでは非実用的です。

**実用的な意味：** フェーズ1・2の液体媒体プロトタイプには、**波長と偏光**符号化のみを使用してください。位相符号化は、能動的な干渉計安定化またはサブミリメートルの光路長が利用可能になるまで延期すべきです。

### 液体媒体の比較

| 媒体 | 透明性 | 屈折率 | dn/dT (K⁻¹) | 推奨用途 |
|---|---|---|---|---|
| 蒸留水 | 200〜1300 nm | 1.333 | −1.0×10⁻⁴ | フェーズ1ベースライン；低コスト |
| グリセロール-水 | 200〜1200 nm | 1.33〜1.47 | −2.5×10⁻⁴ | 調整可能なRI；より高いdn/dT |
| イマージョンオイル | 400〜800 nm | 1.515 | −3.0×10⁻⁴ | RI整合窓；より短い範囲 |
| Fluorinert FC-770 | 250〜2000 nm | 1.275 | −1.7×10⁻⁴ | 最広範囲；最高コスト |

このアプローチは**古典的光学プロトタイプの設計選択**です。単一光子ではなく巨視的な多光子パルスをホストします。量子コヒーレンスや量子優位性を実現しません。

## SCD-CMOS / LED-CMOSそろばん型パターンアーキテクチャ

参照: [docs/scd-cmos-led-pattern-architecture.md](docs/scd-cmos-led-pattern-architecture.md) | [docs/scd-cmos-led-pattern-architecture_ja.md](docs/scd-cmos-led-pattern-architecture_ja.md) | 図解: [diagrams/scd-cmos-led-architecture.md](diagrams/scd-cmos-led-architecture.md) | シミュレータ: [simulator/led_cmos_scd_pattern_demo.py](simulator/led_cmos_scd_pattern_demo.py)

このセクションは、SCDの電子ロジックをLED光パターンとCMOSイメージセンサの読み取りに接続することで、そろばん-OBQCのアナロジーを拡張します。電子十進パターンから光学ビードコンピューティングへの物理的プロトタイプパスを作成します。

### 核心アイデア

そろばんの珠には2つの物理的状態があります：有効または無効。CMOSの配線には2つの電気的状態があります：HighまたはLow。これらは構造的に同一です。したがって、そろばんの桁は5本のCMOS信号線（SCD：Soroban-Coded Decimal）に直接マッピングできます：

```
D = (H, L4, L3, L2, L1)
digit = 5*H + count(L1, L2, L3, L4)
```

この同じオン/オフ構造をLEDに拡張できます。各信号線が1つのLEDを駆動します。CMOSイメージセンサが空間輝度パターンを読み取り、それを桁に復号します。

### 表現レイヤースタック

| レイヤー | 表現 | 役割 |
|---|---|---|
| そろばん | 珠の上/下 | 物理的な十進パターン |
| CMOS SCD | 5本のHigh/Low線 | 電子的な十進パターン |
| LED / RGB LED | 光の位置・色・強度 | 光学パターンジェネレータ |
| CMOSセンサ | ピクセルアレイ | 光学パターンリーダー |
| OBQC | 多自由度光学ビード状態 | 一般化された光学パターンコンピューティング |

### SCD-CMOSであるものとないもの

- **SCDは1桁の十進数あたり5ビットを使用します。** BCDは4ビットを使用します。SCDはBCDよりコンパクトではありません。
- SCDは記憶サイズの削減ではなく、**構造的パターン認識と妥当性チェック**に有用です。
- 下位4ビットはサーモメーターコードを使用します。サーモメーターでないパターンは構造的に無効であり、32通りの可能なパターンのうち22通りは追加のパリティビットなしにエラーとして検出可能です。
- **すべてのエラーが検出可能なわけではありません。** Hビットが反転すると、有効な桁が別の有効な桁に変わる可能性があります（例：0→5, 1→6）。SCDは構造的妥当性コードであり、完全なエラー訂正コードではありません。

### LED-CMOS拡張

LEDディスプレイとCMOSセンサは、純粋なバイナリエレクトロニクスには存在しない光学的な自由度を追加します：色/波長・輝度/強度・空間位置・時間パルスパターン。これはSCDとOBQCを接続します：

- SCDは5本のバイナリ線にわたって構造化された電子珠パターンを使用します
- LED-CMOSはそれらのパターンを光学空間（位置・色・輝度）に拡張します
- OBQCは珠パターンを多次元光学状態（波長・偏光・位相・時間ビン・空間モード）に一般化します

### トイシミュレータ

[simulator/led_cmos_scd_pattern_demo.py](simulator/led_cmos_scd_pattern_demo.py) は以下を実証します：
- SCD桁の符号化
- LED輝度パターンへのマッピング
- ガウス輝度ノイズ注入
- CMOSライクなしきい値読み取り
- 正確 / 検出エラー / サイレントエラーとしての分類

これはトイ輝度ノイズモデルです。物理的な光学シミュレーションではありません。

## 電子回路拡張：そろばん符号化十進ロジック

参照: [docs/electronic-extension-soroban-decimal.md](docs/electronic-extension-soroban-decimal.md) | シミュレータ: [simulator/soroban_decimal.py](simulator/soroban_decimal.py)

そろばんの構造ロジックはフォトニックシステムに限定されません。電子回路設計に**そろばん符号化十進数（SCD）**として直接マッピングされます。これは、デジタルハードウェアにおけるそろばんの棒の物理的構造を反映した5ビット制約十進セルです。

### SCDセル定義

```
D = (H, L4, L3, L2, L1)

digit = 5 * H + count(L4, L3, L2, L1)
```

Hは天珠（値5）であり、L1〜L4は地珠（各値1）であり、隙間なく順番に珠を有効化する**サーモメーターコード**を使用して符号化されます。

### 符号化テーブル

| 十進数 | H | L4 L3 L2 L1 | 5ビットコード |
|---:|:---:|:---:|:---:|
| 0 | 0 | 0 0 0 0 | `0 0000` |
| 1 | 0 | 0 0 0 1 | `0 0001` |
| 2 | 0 | 0 0 1 1 | `0 0011` |
| 3 | 0 | 0 1 1 1 | `0 0111` |
| 4 | 0 | 1 1 1 1 | `0 1111` |
| 5 | 1 | 0 0 0 0 | `1 0000` |
| 6 | 1 | 0 0 0 1 | `1 0001` |
| 7 | 1 | 0 0 1 1 | `1 0011` |
| 8 | 1 | 0 1 1 1 | `1 0111` |
| 9 | 1 | 1 1 1 1 | `1 1111` |

32通りの可能な5ビットパターンのうち有効な状態は10通りです。残りの22通りは定義上**検出可能なエラー状態**です。サーモメーターでない下位パターンはいずれも不正であり、追加のパリティビットなしにフラグを立てることができます。

### 繰り上げと借り下げの構造

インクリメントおよびデクリメント操作はそろばんの機械的ロジックに直接従います：

- **4 + 1：** 下位カウントが4に達し；H = 0 → H = 1にセット，下位をクリア（そろばんの「5の補数繰り上げ」）
- **9 + 1：** H = 1，下位が満杯 → セルを0にリセット，次の桁に繰り上げ
- **5 − 1：** H = 1，下位 = 0 → Hをクリア，下位を1111にセット（5グループからの借り下げ）
- **0 − 1：** セル = 0，下位 = 0 → 9にセット，次の桁から借り下げ

### 光学レイヤーとの関係

SCDは光学ビード状態の**電子的アナログ**です：

| ドメイン | 情報単位 | コンポーネント1 | コンポーネント2 | 有効状態数 |
|---|---|---|---|---|
| 物理的なそろばん | 珠配置 / 棒 | 天珠 H | 地珠（サーモメーター） | 10 |
| SCD（電子） | 5ビット十進セル | Hビット | L1〜L4ビット（サーモメーター） | 32のうち10 |
| OBC（光学） | 光学ビード状態 B | λ, P, φ, ... | 多DOF符号化 | アルファベット定義 |

3つすべてが同じ構造原理を共有しています：値は線形バイナリシーケンスではなく**多成分パターン**です。この整合性により、認知・電子・フォトニックドメインにわたる統一された概念フレームワークが可能になります。

### SCDは記憶密度においてBCDを置き換えることを意図していません。その価値は構造的かつ教育的です：そろばん演算を回路ロジックで直接観察可能にします。

## OBQCではないもの

信頼性のために、以下の明確化が重要です：

- **OBQCは完成した汎用量子コンピュータであるとは主張しません。** 量子ゲート・エラー訂正された量子ビット・量子優位性は実証されていません。
- **OBQCはCPU・GPU・古典コンピュータを汎用計算において上回るとは主張しません。** この段階では比較ベンチマークは存在しません。
- **OBQCはすべての理論的光学状態が確実に識別できるという証明ではありません。** 理論的な状態数は上限です。実用的なアルファベットサイズはより小さく、実験的に決定する必要があります。
- **OBQCはフォトニック多重化スキームではありません。** 光学多重化（WDM・PDM・OFDM）は帯域のために複数の自由度を使用しますが、OBQCはそろばんパターンロジックから着想を得た多値アルファベットとして構造的にそれらを使用します。
- **OBQCは提案フレームワークです。** 明確な実験的ロードマップと反証可能な主張を持つ、そろばん着想多値光学符号化とパターンベース計算を探究するためのものです。

## 反証可能な主張

以下の主張は具体的・測定可能・テスト可能です：

1. **状態符号化精度：** N個の状態を持つ定義された光学ビードアルファベットに対して、指定されたノイズレベルで符号化と最近傍復号がシンボルエラーレートを目標しきい値（例：< 5%）以下に達成できますか？

2. **SERスケーリング：** 固定されたノイズ条件下でアルファベットサイズNが増加するにつれて、シンボルエラーレートはどのように増加しますか？

3. **自由度ロバスト性：** 現実的なノイズ条件下で最高の分離性マージンをもたらすのはどの光学自由度（波長・偏光・位相など）ですか？

4. **冗長性改善：** 冗長符号化（例：多数決投票や繰り返し符号化）を適用すると、シンボルエラーレートが測定可能に低減されますか？

5. **パターン符号化圧縮：** 選択された構造化データタイプに対して、光学ビードパターン符号化はバイナリ符号化と比較して値を表現するために必要なシンボル数を減らしますか？

これらの主張は、ハードウェアが構築される前にソフトウェアシミュレーション（フェーズ0）で最初にテストできます。

## 制約と限界

参照: [docs/limitations.md](docs/limitations.md)

| 制限 | 説明 |
|---|---|
| **ノイズ** | すべての光学システムはショットノイズ・熱ノイズ・増幅器ノイズに晒され、識別可能な状態数を減少させます |
| **クロストーク** | 波長・位相・偏光次元の隣接する状態が相互に漏れ込む可能性があります |
| **光学アライメント** | 位相と偏光の測定には精密なアライメントが必要；機械的擾乱がエラーを引き起こします |
| **検出器の制限** | 検出器の分解能・ダイナミックレンジ・サンプリングレートが達成可能な状態分離性を制約します |
| **スケーラビリティ** | 自由度を追加すると理論上は状態数が乗算的に増加しますが、実際には新しいクロストーク源が導入されます |
| **キャリブレーションドリフト** | 光学システムは熱膨張・機械的緩和・部品の経年劣化により時間とともにドリフトします |
| **エネルギーコスト** | レーザー光源・変調器・極低温検出器（量子拡張用）は相当な電力を消費します |
| **ハードウェアの複雑性** | 多自由度光学システムは精密で高価かつ環境に敏感なコンポーネントを必要とします |
| **理論的vs使用可能な状態数** | 理論的状態積 N = n_λ × n_P × ... は常に実用的な使用可能アルファベットを超えます |
| **実証されたハードウェアなし** | 2026-06-03時点では物理的なプロトタイプは存在しません；すべての結果はソフトウェアシミュレーションによります |
