## 概要（Abstract）

本文書は、そろばんの珠ロジックをCMOSエレクトロニクス、LED光パターン、CMOSイメージセンサによる読み取りに接続することで、光学ビードコンピューティング（OBQC）を拡張するものです。

中心的なアイデアは、そろばんの珠の状態と電子回路のHigh/Lowシグナルが、どちらも物理的なオン/オフ状態であるという点です。10進数の1桁を、5ビットのそろばん型構造パターン（SCD：Soroban-Coded Decimal）として表現できます。このパターンは5本のCMOS信号線に直接割り当て可能であり、LEDの光パターンとして表示し、CMOSイメージセンサで読み取ることができます。

**重要な注意事項：**
- これは汎用コンピュータの置き換えを主張するものではありません。
- SCDはBCDよりもビット効率が高いわけではありません。BCDは4ビット／桁、SCDは5ビット／桁を使用します。
- 追加の1ビットは、省メモリではなく、構造的パターン認識と妥当性チェック、そしてOBQCへの概念的な橋渡しのために使われます。

## そろばんの珠からCMOS信号へ

そろばんの珠には2つの物理的状態があります：有効（桁の値に加算される）と無効（加算されない）。CMOS回路の配線にも2つの電気的状態があります：High（論理1）とLow（論理0）。これらは構造的に等価です。

したがって、上珠（天珠）1個と下珠（地珠）4個からなるそろばんの1列は、5本のCMOS信号線に直接対応します：

| そろばんの要素 | CMOS信号 | 有効時の値 |
|---|---|---|
| 上珠（天珠） | H | 5 |
| 下珠1 | L1 | 1 |
| 下珠2 | L2 | 1 |
| 下珠3 | L3 | 1 |
| 下珠4 | L4 | 1 |

桁の値は次のように計算されます：

```
D = (H, L4, L3, L2, L1)

digit = 5*H + count(L1, L2, L3, L4)
```

下珠はサーモメーターコード方式で符号化されます。有効な珠はL1から順番に隙間なく埋まります。これは、そろばんの地珠が列の下から積み上がる物理的ロジックを忠実に反映しています。

## SCD 符号化テーブル

| 数字 | H | 下珠 | 5ビットパターン |
|---:|---:|---|---|
| 0 | 0 | 0000 | 0 0000 |
| 1 | 0 | 0001 | 0 0001 |
| 2 | 0 | 0011 | 0 0011 |
| 3 | 0 | 0111 | 0 0111 |
| 4 | 0 | 1111 | 0 1111 |
| 5 | 1 | 0000 | 1 0000 |
| 6 | 1 | 0001 | 1 0001 |
| 7 | 1 | 0011 | 1 0011 |
| 8 | 1 | 0111 | 1 0111 |
| 9 | 1 | 1111 | 1 1111 |

32通りの5ビット組み合わせのうち有効な状態は10通りです。残りの22通りはサーモメーター制約に違反する構造的に無効な状態です。

## SCDがBCDとどう異なるか

BCD（Binary-Coded Decimal）は1桁の10進数を4ビットで符号化します。SCDは同じ1桁に5ビットを使います。したがって、**SCDはBCDよりもコンパクトではありません。**

| モデル | 10進1桁あたりのビット数 | 有効状態数 | 無効状態数 | 主な利点 | 主な制約 |
|---|---:|---:|---:|---|---|
| BCD | 4 | 10 / 16 | 6 / 16 | コンパクトな10進符号化 | そろばんとの構造的類似性が低い |
| SCD | 5 | 10 / 32 | 22 / 32 | そろばん型パターンと妥当性チェック | 表現サイズが大きい（25%増） |

SCDは、記憶容量の節約ではなく、そろばんの珠の物理的配置を反映した**構造的パターン**として10進数の1桁を表現するために設計されています。22個の無効状態は、サーモメーター制約の自然な帰結として存在し、追加のパリティビットなしに簡易な妥当性チェックに利用できます。

## エラー検出とサイレントエラー

SCDには32通りのうち22通りの無効状態があるため、多くのランダムなビットパターンを即座に無効と検出できます。ただし、すべてのエラーが検出できるわけではありません。

**検出可能なエラー：**

下位4ビットがサーモメーターパターンでない場合（ビット間に隙間がある場合）は構造的に無効として検出されます：

```
0 0011 -> 0 0101  （L2とL4の間に隙間：無効サーモメーター、検出可能なエラー）
0 1111 -> 0 1011  （隙間あり：検出可能なエラー）
```

**サイレントエラー（検出不可能なエラー）：**

Hビットが反転し、下位ビットが引き続き有効なサーモメーターパターンを維持する場合、結果は別の有効な桁になります。このエラーは検出できません：

```
0 0000 -> 1 0000  （桁0が桁5になる：サイレントエラー）
0 0001 -> 1 0001  （桁1が桁6になる：サイレントエラー）
0 0011 -> 1 0011  （桁2が桁7になる：サイレントエラー）
```

下位ビットがサーモメーター制約を保ったまま変化する場合も同様です：

```
0 0001 -> 0 0011  （桁1が桁2になる：サイレントエラー）
```

**まとめ：** SCDは構造的妥当性コードであり、完全なエラー訂正コードではありません。多くのランダムエラーを検出可能な失敗に変換しますが、特にHビットの反転やサーモメーターを保つ小さな変化によるサイレントエラーは残ります。

完全なエラー訂正には、追加のパリティまたはECCレイヤーが必要です。

## CMOS実装

CMOS実装では、5本のSCD信号線を5本の独立したビットとしてではなく、1つの10進セルとして扱うことができます。これにより、回路構造がそろばんの列という抽象概念と対応します。

### 構成要素

- **1桁あたり5本の信号線：** H、L1、L2、L3、L4
- **妥当性チェッカー：** サーモメーター制約を確認する組み合わせ論理回路。{0000, 0001, 0011, 0111, 1111}以外の下位ビットパターンを検出します。シンプルなゲートアレイで実装可能です。
- **サーモメーターパターン検出器：** 妥当性チェッカーのサブ回路。5つの有効な下位パターンを識別します。
- **桁デコーダー：** 有効なSCDパターンを4ビットBCDまたは2進数出力に変換します。
- **インクリメントロジック：** そろばんの繰り上げルールに従います。H=0で下珠が1111になったらH=1にして下珠をクリア。H=1で下珠が1111になったら次の桁に繰り上げ。
- **デクリメントロジック：** そろばんの借り下げルールに従います。H=1で下珠が0000のとき、Hをクリアして下珠を1111にセット。
- **繰り上げ/借り下げの伝播：** 複数桁の演算のために桁セル間でカスケード接続します。
- **オプションECCレイヤー：** サイレントエラーのカバレッジのための追加パリティまたは冗長符号化。

妥当性チェッカーは外部の参照状態を必要としません。5本の信号線上の信号値のみを使って適法性を判定するため、組み合わせCMOSロジックでシンプルに実装できます。

## LED-CMOS 光学拡張

LEDディスプレイとCMOSイメージセンサは、SCD電子ロジックから光学ビードコンピューティングへの直感的な橋渡しを提供します。そろばんの珠をCMOS配線にマッピングするのと同じオン/オフ構造を、LED光源とセンサピクセルに拡張できます。

### LED側

1桁あたり5個のLEDがSCDパターンを直接表現できます。各LEDは1本の信号線に対応します。光学符号化への拡張により、さらに多くの自由度が加わります：

- **位置：** アレイのどのLEDが有効か
- **色 / 波長：** 異なるLED色で異なる珠の種類や役割を表現
- **輝度 / 強度：** 1つのLEDあたり複数の識別可能な輝度レベル
- **時間的点滅：** 追加の状態情報を持つパルスパターン
- **RGBチャンネル：** 1つのLEDあたり3つの独立したカラーチャンネル
- **マイクロLEDアレイ：** 高密度な空間符号化

### CMOSセンサ側

CMOSイメージセンサはLEDアレイが発する空間光パターンを読み取ります：

- 各ピクセルの入射光強度を電子信号に変換
- 色、強度、位置の状態を分類
- しきい値または分類モデルを適用して受信パターンをデコード
- チャンネルごとの配線を必要とせずに多チャンネルパターンリーダーとして機能

### 電気光学プロトタイプのパス

```
SCD デジタル状態
-> LED / RGB LED / マイクロLED 光パターン
-> 光学媒体（空気、密閉セル、導波路）
-> CMOSイメージセンサ
-> パターン分類器（最近傍法、CNN、またはしきい値処理）
-> デコードされたSCD状態またはOBQC状態
```

## LED色が重要な理由

単純なオン/オフ電子回路は、1配線あたり1ビットしか運びません。光には、純粋な電子バイナリシステムには存在しない追加の自由度があります：

| 自由度 | 電子的アナログ | 光学的拡張 |
|---|---|---|
| オン/オフ | High/Low | LED点灯/消灯 |
| — | （利用不可） | 色 / 波長 |
| — | （利用不可） | 輝度 / 強度 |
| — | （利用不可） | アレイ内の空間位置 |
| — | （利用不可） | パルスタイミング / 時間パターン |
| — | （利用不可） | 偏光（追加光学素子使用時） |

これがSCDとOBQCを結びつけます。SCDは5本のバイナリ信号線に構造的な電子珠パターンを使います。OBQCはその珠パターンを光学的な自由度（波長、偏光、位相、時間ビン、空間モード）に一般化します。LED-CMOSレイヤーは、電子SCDと完全な光学ビードコンピューティングの間にある物理的な中間層です。

## 順次計算ではなくパターン認識

SCDとOBQCのより深い概念的な転換は、シンボリックなバイナリ操作から構造的なパターン認識への移行です。

**従来のデジタル計算：**

```
数値 -> 2進表現 -> 逐次論理演算 -> 結果
```

**そろばん型パターン計算：**

```
数値 -> 珠パターン -> パターン認識 -> パターン変換 -> 結果
```

これはすべての計算を置き換えるものではありません。以下の場合に有用かもしれません：

- 10進数の構造が自然に重要な場合（金融、計数、表示アプリケーション）
- 高速な構造的検証がコンパクトな記憶より重要な場合
- 視覚的または光学的な入力が自然なインターフェースの場合（カメラ読み取り、LEDアレイ）
- パターン分類がワークロードの中心の場合
- センサ、LED、CMOSシステムがハードウェアにすでに存在する場合

フラッシュ暗算（フラッシュ暗算）は、パターンベースの10進数処理に認知的な先例があることを示しています。熟練した実践者は桁シンボルではなく、空間的なそろばんパターンイメージを操作します。LED-CMOS SCDアーキテクチャは、この認知モデルの工学的アナログです。

## プロトタイプアーキテクチャ

### 電子SCDプロトタイプ

```
SCDエンコーダ
  -> CMOS妥当性チェッカー（サーモメーター制約）
  -> 桁デコーダ
  -> 繰り上げ/借り下げロジック
  -> 複数桁演算ユニット
```

### 電気光学プロトタイプ

```
SCDエンコーダ
  -> LEDマトリクスドライバ
  -> LEDアレイ（1桁あたり5個のLED、オプションでRGB）
  -> 光学媒体（空気または密閉セル）
  -> CMOSカメラ / イメージセンサ
  -> パターン分類器（最近傍法またはしきい値処理）
  -> デコードされたSCD状態
```

### 光学ビードプロトタイプ

```
RGB LED / マイクロLEDアレイ
  -> 光学媒体（空気、液体セル、導波路）
  -> CMOSセンサアレイ
  -> （波長、輝度、位置）に基づく最近傍分類器
  -> OBQC状態
```

## 反証可能な実験

このフレームワークを評価するために実施可能な実験：

1. **BCD対SCDエラー検出比較：** BCDとSCDの符号化にランダムな独立ビット反転を適用します。無効状態検出率とサイレントエラー率を測定します。SCDがより多くのエラーを検出することを確認します（無効状態の割合が大きいため）。

2. **サイレントエラーの特性評価：** 各SCD桁に対して、1ビットおよび2ビット反転の結果をすべてマッピングします。サイレントである（有効状態から有効状態への遷移）反転を特定します。

3. **LED-CMOS桁リーダー：** 1つのSCD桁用に5個のLEDからなる物理的なディスプレイを構築します。CMOSカメラで読み取ります。通常の照明条件下で10個の有効パターンすべてを正しく分類します。

4. **ノイズ下での分類：** 輝度の変動、ぼかし、環境光、カメラノイズを追加します。ノイズレベルの関数として分類精度を測定します。

5. **RGB色拡張：** バイナリLEDのオン/オフをRGBカラー符号化に置き換えます。分類器が色情報を使ってLED位置あたり2状態以上を識別できるかテストします。

6. **パターン認識の演算コスト：** 特定のワークロードにおいて、パターンベースのSCD分類と逐次BCDビットチェックの演算コストを比較します。

7. **センサノイズモデルの検証：** トイLED-CMOSシミュレータの結果と物理的なLED-カメラ計測を比較します。

## 制約と限界

- **SCDは1桁あたり5ビットを使用します。** これはBCDより25%多く、複数桁の数値の表現サイズが増加します。
- **妥当性チェックは完全な訂正ではありません。** Hビット反転やサーモメーターを保つ下位ビット変化によるサイレントエラーは残ります。完全なエラー訂正には追加のECCレイヤーが必要です。
- **LED/CMOSシステムは現実のノイズを伴います：** 輝度変動、焦点エッジのぼかし、環境光汚染、カラーチャンネルのクロストーク、センサ量子化、レンズ歪みなどが分類精度を低下させます。
- **キャリブレーションドリフト：** LEDは温度と経年変化で輝度が変化します。CMOSセンサにはダーク電流ドリフトと画素不均一性があります。定期的な再キャリブレーションが必要です。
- **光学プロトタイプは専用CMOSロジックより低速です**（マイクロLEDアレイ、FPGA接続センサ、カスタムASICなどの専用ハードウェアを使用しない限り）。
- **汎用コンピュータ置換は主張しません。** SCD-CMOSとLED-CMOSはパターン指向の10進数ツールであり、汎用演算エンジンではありません。
- **これは実証されたシステムではなく、探索のためのフレームワークです。** 関連シミュレータのすべての結果は、明示的に述べられた前提条件のもとでのトイモデルです。

## まとめ

SCD-CMOSおよびLED-CMOSアーキテクチャは、そろばんの珠ロジック、電子的なオン/オフ状態、および光学的なパターン認識を橋渡しします。

その価値はコンパクトな記憶にありません。SCDはBCDよりも多くのビットを使います。その価値は：

1. **構造的表現：** 桁パターンがそろばんの列の配置を直接反映します
2. **妥当性チェック：** 32通りのうち22通りの無効状態が追加パリティなしで検出可能です
3. **光学的橋渡し：** 同じオン/オフ構造をLEDの位置、色、強度に拡張できます
4. **OBQCへのパス：** LED-CMOSプロトタイピングは、電子SCDから多次元光学状態符号化を持つ完全な光学ビードコンピューティングへの物理的に実現可能な橋渡しを提供します

SCD-CMOSは構造的な10進パターンレイヤーです。LED-CMOSはその光学インターフェースです。OBQCはそれを多次元光学ビード状態へと一般化したものです。
